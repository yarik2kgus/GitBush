2.Отправляем публичный ключ в настройки своего аккаунта в git hub

3.git config --global user.name “Vitali Shulha“
  git config --global user.email “vitali_Shulha@epam.com" - создаем имя и почту для того что бы было видно кто конкретно взаимодействует с файлами и делает комиты

4.cd /D или C/название папки/ - Заходим в папку в окторой будем работать

5.git clone ssh ссылка на клонирование взятая из github - Клонируем файлы  с репозитория в выбранную ранее папку 

Основные команды:

git status - проверка статуса папки на новые файлы и тд

git add название файла - подготовка к коммиту то есть выбираем из новых файлов те, которые будут включены в конечный коммит

git commit - коммитит все файлы выбранные в add , так же можно при помощи флага -m "туекст" добавить комментарий к коммиту (сохраняет изменения лишь локально, используется что бы создавать "точки сохранени")

git log - проверка истории коммитов

git push - отправдяет все мои локальные коммиты с git commit  на репозиторий

Далее:

git remote -v - для проверки на каком мы сайте с репозиторием (проверка на гитхабе ли мы)

pwd - проверка в какой папке мы сейчас находимся
